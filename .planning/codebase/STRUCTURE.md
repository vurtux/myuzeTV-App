## Directory Purposes

**`app/` (Expo Router Routes):**
- Purpose: File-based routing with dynamic segments
- Contains: Route handlers (screen components), layouts
- Key files: `_layout.tsx` (root), `(tabs)/_layout.tsx` (tabs), dynamic routes
- Naming: `index.tsx` for route defaults, `[param]` for dynamic segments, `(group)` for route groups

**`components/` (Reusable UI):**
- Purpose: React Native components, screens, modals
- Contains: 30+ components including:
  - Screens: `LoginScreen`, `ProfileScreen`, `DramaDetailScreen`, `SubscribeScreen`
  - Cards: `WatchlistCard`, `DramaCard` (via rails/hero)
  - Rails: `DramaRail`, `ContinueWatchingRail` (horizontal scrolls)
  - Player: `EpisodeFeed`, `SingleEpisodePlayer` (vertical player)
  - Overlays: `GenreChips`, `SearchOverlay`, `EpisodeDrawer`
  - Layout: `BottomNav`, `TopBar`
  - Utilities: `SkeletonBox`, `SkeletonHero`, `SkeletonRail`, `LoadingButton`
  - Images: `DramaImage.tsx` (native), `DramaImage.web.tsx` (web)
- Styling: NativeWind/Tailwind classes via `className` prop

**`api/` (HTTP Client & Endpoints):**
- Purpose: Server communication and response mapping
- Contains:
  - `client.js` → axios instance with auth + error interceptors
  - `dramas.js` → `fetchDramas()`, `fetchDramaDetail()`, `fetchContinueWatching()`, `saveProgress()`
  - `rails.js` → `fetchRails()` (curated content, sorted by display_order)
  - `watchlist.js` → `fetchWatchlist()`, `addToWatchlist()`, `removeFromWatchlist()`
  - `episodes.js` → episode-level endpoints
  - `subscription.js` → subscription check/purchase
  - `account.js` → user profile endpoints
  - `auth.js` → backend auth endpoints
  - `mappers.js` → `resolveImageUrl()`, `placeholderFor()` (field mapping + image resolution)
- Pattern: Each endpoint file exports async functions that call `api.get()` or `api.post()` and map response

**`context/` (React Context Providers):**
- Purpose: Global state and shared functionality
- Contains: `AuthContext.tsx` (authentication, subscription state, auth methods)
- Provides: `useAuth()` hook for access to `{ user, token, isAuthenticated, isSubscribed, signInWithGoogle, sendPhoneOtp, verifyPhoneOtp, signOut, setGuestMode }`

**`hooks/` (Custom React Hooks):**
- Purpose: Reusable stateful logic
- Contains:
  - `usePlaybackRequest()` → handles auth + subscription checks before playback
  - `useWatchlist()` → manages watchlist state via React Query mutations
  - `usePressAnimation()` → animated button press feedback
- Pattern: Hooks return data + methods; mutations invalidate React Query cache on success

**`lib/` (Utilities, Types, Services):**
- Purpose: Shared utilities not tied to UI
- Auth & Storage:
  - `auth-storage.js` → `getApiKey()`, `getUserToken()`, `setUserToken()`, `getAuthToken()`
  - `auth-invalid.js` → 401 callback mechanism
  - `auth-service.js` → Firebase auth methods (Google sign-in, phone OTP)
  - `firebase.js` → Firebase app initialization
- Types:
  - `types.ts` → `Drama` interface (id, title, image, genre, progress, etc.)
  - `drama-detail-types.ts` → `Episode`, `DramaDetail` interfaces
  - `drama-data.ts` → shared drama data types
- Services:
  - `analytics-service.ts`, `.web.ts`, `.native.ts` → Mixpanel, MoEngage, Firebase Analytics
  - `date-utils.ts` → date formatting helpers

**`__tests__/` (Jest Unit Tests):**
- Purpose: Component and utility testing
- Contains: `api/`, `components/`, `hooks/`, `lib/` mirroring source structure
- Naming: `.test.tsx` or `.test.js` files
- Config: Jest preset `jest-expo/web`, ignores `v0-reference/` and `e2e-playwright/`

**`e2e-playwright/` (Playwright E2E Tests):**
- Purpose: Critical user flow testing (login, home, profile, player, search)
- Contains: `.spec.ts` files targeting mobile Chrome (Pixel 5 viewport)
- Config: `playwright.config.ts` auto-starts Expo web on port 8082
- Run: `npm run test:e2e` or `npm run test:e2e:headed`

**`assets/` (Images & Branding):**
- Purpose: App icons, splash screens, branding assets
- Contains: `icon.png`, `splash-icon.png`, `adaptive-icon.png`, `favicon.png`, `myuzeTV.png`

**`scripts/` (Build & Deployment):**
- Purpose: Automation for build, deploy, config injection
- Contains: `inject-config.js` (injects `window.__MYUZE_AUTH_TOKEN__` into web builds)

**`dist/` (Web Build Output):**
- Purpose: Static export of web build
- Generated by: `npm run build:web` (Expo export)
- Contains: `index.html`, static JS/CSS chunks, `config.js` (injected by `inject-config.js`)

**`coverage/` (Jest Coverage):**
- Purpose: Test coverage reports
- Generated by: `npm test` with coverage flag
- Contains: `lcov-report/`, `coverage.json`

**`v0-reference/` (Design Reference):**
- Purpose: Original Next.js v0 prototype (not imported by app)
- Excluded from: Jest tests, build, runtime
- Usage: Visual reference only for design decisions

- `global.css` → Tailwind config, CSS custom properties, global styles

**Core Logic:**
- `context/AuthContext.tsx` → Authentication state, dual-token system, test accounts

- `api/mappers.js` → Image URL resolution, placeholder generation
- `hooks/usePlaybackRequest.ts` → Auth + subscription checks before playback
- `hooks/useWatchlist.ts` → Watchlist mutations with cache invalidation
- `lib/auth-storage.js` → Token management (AsyncStorage/SecureStore)
- `lib/auth-invalid.js` → 401 callback mechanism

**Testing:**
- `__tests__/api/` → API client and endpoint tests
- `__tests__/components/` → Component render and interaction tests
- `__tests__/hooks/` → Hook logic tests
- `e2e-playwright/` → Critical user flow E2E tests

## Naming Conventions

**Files:**
- Route files: `index.tsx` (defaults), `[param].tsx` (dynamic), `_layout.tsx` (layouts)
- Component files: `PascalCase.tsx` (e.g., `LoginScreen.tsx`, `DramaRail.tsx`)
- Utility/service files: `camelCase.js` or `camelCase.ts` (e.g., `auth-storage.js`, `drama-data.ts`)
- Test files: `{name}.test.tsx` or `{name}.test.js`
- E2E tests: `NN-{description}.spec.ts` (e.g., `01-login.spec.ts`)
- Platform-specific: `{name}.web.tsx`, `{name}.native.ts` (Expo convention)

**Directories:**
- Feature/domain dirs: lowercase with hyphens (e.g., `(tabs)`, `api/`, `hooks/`)
- Component dirs: `components/` (flat, no subdirs; 30+ files)
- Test dirs: `__tests__/` (mirrors source structure)
- Build output: `dist/`, `coverage/`, `playwright-report/`

**Type Definitions:**
- Interfaces: `PascalCase` (e.g., `User`, `Episode`, `DramaDetail`)
- Unions: `camelCase` suffix with `Type` or `Status` (e.g., `subscriptionStatus`, `episodeStatus`)
- Generics: Single letter or descriptive (e.g., `T`, `TData`)

## Where to Add New Code

**New Feature:**
- Primary code: 
  - Route handler: `app/[feature]/[param].tsx`
  - Component: `components/[FeatureName].tsx`
  - API endpoint: `api/[feature].js` with mapping functions
  - Context (if global state): `context/[Feature]Context.tsx`
  - Hook (if reusable logic): `hooks/use[Feature].ts`
- Tests: `__tests__/[type]/[name].test.tsx`
- E2E: `e2e-playwright/NN-[feature].spec.ts`

**New Component/Module:**
- Implementation: `components/[ComponentName].tsx` (if UI) or `lib/[module].ts` (if utility)
- Props interface: Define above component, name `[Component]Props`
- Styling: Use NativeWind/Tailwind via `className`
- Export: Named export + default export in index (if barrel file exists)

**Utilities:**
- Shared helpers: `lib/` for non-UI utilities (auth, date formatting, types)
- API mappers: `api/mappers.js` (field resolution, image URLs)
- Custom hooks: `hooks/use[Feature].ts` (React hooks for state/side effects)
- Constants: `constants/` (hardcoded values, config)

## Special Directories

**`(tabs)` Route Group:**
- Purpose: Grouped routes sharing `BottomNav` layout
- Files: `_layout.tsx` (tab layout), `index.tsx` (home), `watchlist.tsx`, `profile.tsx`
- How it works: Expo Router treats `(tabs)` as a non-path segment; all routes in group share the layout

**`.env` & Config:**
- Purpose: Runtime configuration (API base URL, CDN, auth token)
- Files: `.env` (development), `.env.production` (production)
- Required vars: `EXPO_PUBLIC_API_URL`, `EXPO_PUBLIC_CDN_URL`, `EXPO_PUBLIC_AUTH_TOKEN`
- Not committed: `.env` files are in `.gitignore`
- Web deploy: `scripts/inject-config.js` injects `window.__MYUZE_AUTH_TOKEN__` into HTML (no env vars on web)

**`dist/` (Web Build Output):**
- Purpose: Static web build for deployment
- Generated: `npm run build:web` → Expo export
- Contents: `index.html`, JS chunks, CSS chunks, `config.js`
- Not committed: Regenerated on each deploy
- Deployment: Bunny CDN or static hosting (see `BUNNY_DEPLOY.md`)

**`coverage/` (Test Coverage):**
- Purpose: Jest coverage reports
- Generated: `npm test -- --coverage`
- Not committed: Can be gitignored
- Targets: 80%+ coverage for quality phases

## Import Path Aliases

**TypeScript/Expo:** No path aliases configured; use relative imports (`../lib/`, `../../api/`)

**Recommendation for future:** Add `paths` in `tsconfig.json`:
```json
{
  "compilerOptions": {
    "paths": {
      "@/*": ["*"],
      "@/components": ["components"],
      "@/api": ["api"],
      "@/lib": ["lib"],
      "@/hooks": ["hooks"],
      "@/context": ["context"]
    }
  }
}
```
